# 🧠 Experience OS - 个人经验自动结构化系统

## 一、项目概述

### 1.1 项目名称
**Experience OS（体验操作系统）**

### 1.2 一句话定义
一个用 AI 将日常"零散生活观察与行为实验"自动沉淀为可复用经验规则的个人经验系统。

### 1.3 项目背景
基于6月30日公司AI自动化场景创意比赛要求，本项目旨在通过AI技术实现个人生活经验的自动结构化，帮助用户将零散观察转化为可行动的决策模型。

---

## 二、核心问题分析

### 2.1 用户痛点
| 痛点编号 | 痛点描述 | 影响分析 |
| :--- | :--- | :--- |
| P01 | 经验碎片化：记录在脑子里/备忘录/容易忘记 | 知识资产流失 |
| P02 | 无法复用：下次遇到同样问题还是重新试 | 效率低下 |
| P03 | 没有结构：分不清"规律"与"偶然" | 决策质量低 |

### 2.2 用户场景
用户每天在做很多"小实验"：
- 什么时间买东西最好吃
- 什么方式最省时间
- 什么习惯最舒服
- 哪种选择更有效

## 三、系统能力拆解

### 3.1 能力架构
```
┌─────────────────────────────────────────────────────┐
│                   Experience OS                     │
├─────────────────────────────────────────────────────┤
│  1. 低成本记录（输入层）                            │
│     └── 一句话记录，无需结构化                       │
├─────────────────────────────────────────────────────┤
│  2. AI自动整理（核心层）                            │
│     ├── 分类：饮食/购物/生活/时间/偏好              │
│     ├── 提炼规律：最佳时间窗口、影响因素、成功条件    │
│     └── 去重&关联：历史记录匹配、趋势形成            │
├─────────────────────────────────────────────────────┤
│  3. 经验沉淀（输出层）                              │
│     ├── 可复用规则                                 │
│     ├── 策略卡片                                   │
│     └── 地图化/时间轴（扩展）                       │
└─────────────────────────────────────────────────────┘
```

### 3.2 系统闭环
```
观察 → 记录 → AI归纳 → 生成规律 → 下次使用 → 再优化
```

## 四、功能需求（MVP版本）

### 4.1 输入模块
| 需求编号 | 需求描述 | 优先级 |
| :--- | :--- | :--- |
| FR01 | 用户可通过输入框提交一句话观察/体验 | HIGH |
| FR02 | 支持快速记录，无需复杂表单填写 | HIGH |
| FR03 | 记录时可选择添加时间/地点标签（可选） | MEDIUM |

### 4.2 AI处理模块
| 需求编号 | 需求描述 | 优先级 |
| :--- | :--- | :--- |
| FR04 | AI自动对记录进行分类（饮食/购物/生活/时间/偏好） | HIGH |
| FR05 | AI提炼规律：识别最佳时间窗口、影响因素、成功条件 | HIGH |
| FR06 | AI自动去重：检测与历史记录的相似度 | HIGH |
| FR07 | AI关联分析：发现记录之间的趋势和模式 | MEDIUM |

### 4.3 输出展示模块
| 需求编号 | 需求描述 | 优先级 |
| :--- | :--- | :--- |
| FR08 | 经验列表视图：展示所有记录及AI总结 | HIGH |
| FR09 | 规则库视图：展示可复用策略集合 | HIGH |
| FR10 | 卡片展示：结论+推荐+是否可复用标识 | HIGH |
| FR11 | 经验地图视图：地点/时间/行为分布可视化 | MEDIUM |

## 五、非功能需求

### 5.1 必须满足
| 需求编号 | 需求描述 | 说明 |
| :--- | :--- | :--- |
| NFR01 | 记录极轻量 | 用户只需写一句话，不超过50字 |
| NFR02 | AI负责结构化 | 用户不参与手动分类整理 |
| NFR03 | 输出可行动 | 规则必须是具体可执行的建议 |
| NFR04 | 响应速度 | AI处理延迟 < 3秒 |
| NFR05 | 数据安全 | 个人数据本地存储，不云端同步 |

### 5.2 不做范围
- 长日志系统
- 手动分类系统
- 重操作笔记工具

## 七、比赛展示方案

### 7.1 演示流程
1. **开场（1分钟）**：介绍痛点和项目价值
2. **输入演示（2分钟）**：现场输入一条生活观察
3. **AI处理（2分钟）**：展示AI分类和规律提炼过程
4. **结果展示（2分钟）**：展示生成的策略卡片和规则库
5. **价值总结（1分钟）**：总结系统带来的效率提升

### 7.2 展示亮点
- ✨ 极轻量输入：一句话记录
- ✨ AI自动处理：无需人工干预
- ✨ 可行动输出：直接指导行为
- ✨ 持续优化：闭环反馈机制
