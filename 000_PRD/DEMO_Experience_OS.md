# Experience OS 比赛演示脚本

## 1. 演示目标

用 8 分钟证明一件事：Experience OS 可以把零散生活观察，自动整理成下一次可复用的个人经验规则。

本次演示只讲 MVP 闭环，不讲账号、云同步、真实地图 SDK、复杂推荐算法。

## 2. 演示前准备

- 打开 H5：`http://localhost:5173/`
- 点击「载入演示数据」
- 确认顶部数据不为 0，规则库里有多类规则
- 保留一条现场输入样例，用来展示即时生成能力

推荐现场输入：

> 周末10点健身房人少，器械不用排队

## 3. 8 分钟流程

| 时间 | 页面 | 讲解重点 | 操作 |
| --- | --- | --- | --- |
| 0:00-1:00 | 首页 | 每个人都有大量经验，但通常散落在记忆、备忘录和聊天里 | 说明痛点 |
| 1:00-2:00 | 首页 | Experience OS 的定义：一句话记录，AI 自动变成规则 | 展示输入框和样例 |
| 2:00-3:30 | 最新策略卡 | AI 提取分类、结论、推荐行动、适用条件和注意事项 | 输入现场样例并生成 |
| 3:30-4:40 | 经验列表 | 原始观察不会丢失，AI 摘要和标签会形成可追溯记录 | 切到「经验」 |
| 4:40-5:50 | 规则库 | 多条观察沉淀成可复用规则，支持分类和搜索 | 切到「规则库」并筛选 |
| 5:50-6:40 | 策略卡 | 用户反馈让系统知道哪些规则有用、待观察或不准确 | 点一次「有用」或「待观察」 |
| 6:40-7:30 | 地图/时间轴 | 地点聚合和经验演化是扩展亮点，但不依赖真实地图 | 展示「地图」和「时间轴」 |
| 7:30-8:00 | 首页/规则库 | 总结：AI 从聊天回答变成个人经验整理系统 | 回到规则库收尾 |

## 4. 讲解话术

开场：

> 我们每天都会积累很多经验，比如什么时候去健身房人少、雨天走哪条路更近、什么时间写方案效率更高。但这些经验通常只停留在脑子里，下一次遇到相似场景时不一定想得起来。

产品定义：

> Experience OS 做的事情很简单：用户只写一句观察，系统自动识别分类、条件、结论和推荐行动，把它沉淀成下一次可复用的个人经验规则。

闭环总结：

> 这个 MVP 的核心不是记日记，而是把个人观察变成结构化规则，并通过反馈逐步区分“有用规则”和“待观察经验”。

## 5. 现场检查

如果现场输入没有形成稳定规则，说明输入证据不足，展示“待观察”状态。

如果本地数据被清空，点击「载入演示数据」重新预置演示记录。

如果预览服务未启动，执行：

```bash
npm run preview:h5 -- --port 5173
```

如果需要重新构建，执行：

```bash
npm run build:h5
```

## 6. 不讲内容

- 不承诺真实地图导航能力
- 不承诺多端同步和账号体系
- 不把单条观察包装成绝对规律
- 不展示伪精确置信度数字
