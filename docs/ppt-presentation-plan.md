## 三、每页详细内容

> **主线故事:** 围绕"工作效率"主题贯穿三大亮点,降低评委认知负担。

### P1 - 封面

**标题:** Experience OS
**副标题:** 个人经验管理与决策辅助引擎
**Slogan:** 让每一次经历,都变成下一次决策的资产
**底部:** 参赛信息 / 演讲人 / 日期

**讲解词:**
> "大家好,今天介绍 Experience OS——一个个人经验管理与决策辅助引擎。我们的目标很简单:让每一次经历,都变成下一次决策的资产。"

---

### P2 - 痛点

**标题:** 经验管理的三重困境

**三栏对比:**

| 记笔记 | 建知识库 | 用 AI 助手 |
|--------|---------|-----------|
| 记了就忘 | 结构太重 | 问完就散 |
| 需要手动整理 | 维护负担大 | 没有沉淀 |
| 无法主动召回 | 无法跨记录发现规律 | 经验不可复用 |

**讲解词:**
> "我们都在记录经验,但现有工具有三个问题:记笔记记了就忘,建知识库结构太重,问 AI 助手问完就散。共同的问题是——经验没有被结构化沉淀,更没有主动召回。"

---

### P3 - 定位

**标题:** 介于笔记和知识库之间

**核心定位图:**

```
笔记 ──────── Experience OS ──────── 知识库
记录事件         沉淀规律            组织知识
被动存储         AI 自动提炼         手动维护
需要查找         主动召回            体系化检索
```

**一句话定位:**
> "不是日记,不是知识库——是**经验引擎**:一句话输入,AI 自动提炼规律,决策时主动召回。"

**讲解词:**
> "Experience OS 介于笔记和知识库之间。比笔记更智能——AI 自动提炼规律;比知识库更轻量——一句话输入,无需结构化模板。核心是让经验从'记录'升级为'可复用的决策资产'。"

---

### P4 - 核心闭环

**标题:** 核心闭环:从观察到决策

**流程图:**

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  一句话   │ →  │  AI 自动  │ →  │  规律发现  │ →  │  决策召回  │
│  观察    │    │  提炼    │    │  跨记录   │    │  主动建议  │
│          │    │  规则    │    │  聚类    │    │  可信度   │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
   输入            沉淀            进化            应用
```

**技术架构(集成在同一页底部):**

```
┌─────────────────────────────────┐
│  UI 层:Vue 3 + Pinia + Vite    │
├─────────────────────────────────┤
│  智能层:AI 提炼 / 规律发现 / 决策 │
│  ↓ 模型不可用时降级到本地算法     │
├─────────────────────────────────┤
│  存储层:localStorage(本地优先)  │
└─────────────────────────────────┘
```

**三大原则(底部一行):**
- **Local First** · **Model Agnostic** · **Security Contract** | 39+ 单元测试覆盖核心逻辑

**讲解词:**
> "核心闭环四个环节:用户一句话输入观察,AI 自动提炼成结构化规则,多条相似观察聚合成规律,未来决策时主动召回相关经验。技术上坚持本地优先、模型无关、安全契约,核心逻辑有 39+ 单元测试覆盖,保证可靠性。"

---

### P5 - 亮点一:AI 提炼

**标题:** 一句话观察 → 结构化经验

**示例(工作效率主线):**

**输入(用户):**
> "下午3点写方案效率最高"

**AI 自动产出:**
- 类别:工作
- 方向:策略(strategy)
- 标题:高效写作时段策略
- 结论:下午3点左右是方案写作的高效时段
- 推荐行动:将重要写作任务安排在下午3点
- 信任度:待验证

**讲解词:**
> "用户只需说一句口语化的话,AI 自动完成分类、提炼结论、给出推荐行动,并打上信任度标签。整个过程中用户不需要任何结构化输入。"

---

### P6 - 亮点二:规律发现

**标题:** 跨记录语义聚类 → 规律库

**核心机制:**
- 两段式聚类:统计粗分 + 模型语义归因
- 跨簇合并:不同表述、同根因 → 一条规律
- 生命周期:active → reviewed → resolved
- 时间维度:复发次数、趋势(Mann-Kendall 检验)、置信度

**示例(工作效率主线延续):**
```
观察1: "下午3点写方案效率最高"
观察2: "上午10点开会容易走神"
观察3: "晚上加班写代码容易出错"
           ↓ 语义聚类
规律: "工作效率与时段强相关"(复发3次,趋势稳定,置信中)
```

**讲解词:**
> "多条表面不同但同根因的观察,系统会自动聚合成一条规律,并标注复发次数、趋势和置信度。规律有生命周期——可以标记'已针对它改进',遇到反例还能自动修正。"

---

### P7 - 亮点三:决策召回

**标题:** 决策时主动召回相关经验

**场景示例(工作效率主线收尾):**
- 用户输入:"明天要写方案"
- 系统召回:"你记过'下午3点写方案效率最高'"
- 可信度标注 + 建议:建议安排在下午3点

**三层能力:**
1. **关键词召回** + 模型语义匹配
2. **可信度信号**(trustSignal):基于历史验证记录
3. **建议合成**:本地确定性建议 + opt-in AI 润色

**讲解词:**
> "当用户面临决策时,系统主动召回相关历史经验,并基于可信度给出建议。比如你说明天要写方案,系统会提醒你之前记过'下午3点效率最高'——建议安排在那个时段。一条观察就能触发召回,无需用户完美记录。"

## 六、Q&A 预案

### 常见问题与应答

| 问题 | 应答要点 |
|------|---------|
| **和 ChatGPT 的记忆功能有啥区别?** | ChatGPT 记忆是"对话上下文",我们是"结构化经验库"——有分类、置信度、生命周期,可主动召回 |
| **用户凭什么持续记录?** | 输入成本极低(一句话)+ AI 自动提炼价值 + 决策时主动召回让用户感受到回报 |
| **规律发现的准确率如何验证?** | 三重防护:最少成员数 + 置信度分级 + 生命周期可纠偏 + 反例自动修正(V3 trustSignal) |
| **和 Notion/印象笔记有啥区别?** | 它们是"存储工具",我们是"智能引擎"——自动提炼 + 规律发现 + 主动召回 |
| **模型挂了怎么办?** | 本地降级链:关键词 + 统计聚类,功能不中断 |
| **数据安全?** | 本地优先,API Key 不上云,模型调用前自动脱敏(PII 过滤) |
| **怎么商业化?** | V5 组织经验:团队经验库 + 飞书生态 |
