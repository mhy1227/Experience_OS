# A7 实施方案:趋势检验(Mann-Kendall)

> 日期:2026-06-23 · 状态:实施方案(纯方案,未编码)· 锦上添花,体量小
> 关联:`docs/algorithm-upgrade-plan.md`(A7)、`services/lawDiscovery.ts`(`Law.trend`)
> 目标:把 `Law.trend`(上升/平/下降)从"拍脑袋阈值比"换成**统计趋势检验**。

---

## 0. 修的痛
现状:`Law.trend` 用"近 30 天占比 vs 前 60 天"的阈值比——魔数、小样本不稳。

## 1. 方法:Mann-Kendall 趋势检验
非参数趋势检验,**小样本、不要求正态分布**,适合复发次数这种离散少量数据。

- 把成员按时间分桶,得序列 `x₁..xₙ`(每桶复发数);
- 统计量 **S = Σ_{i<j} sign(xⱼ − xᵢ)**(所有点对比较:后比前大记 +1,小记 −1,平 0);
- **S 显著为正 → rising;显著为负 → falling;否则 flat**;
- 小样本可用 S 的精确分布判显著;数据稍多用标准化 Z。

## 2. 备选(更简单)
时间桶上做**线性回归**,看斜率正负 + 显著性。比 MK 更直观,精度略低。二选一即可。

## 3. 模块 + 接入
- `src/services/trend.ts`(纯函数):`mannKendall(series): 'rising'|'flat'|'falling'`;
- 接入:`lawDiscovery` 算 `trend` 处替换阈值比;阈值/显著水平进 config(别留魔数)。

## 4. 测试设计(node:assert)
1. 单调增序列 → rising;单调减 → falling;无序/平 → flat;
2. 极小样本(n≤2)→ flat,不抛错;
3. 全相等 → flat(S=0)。

## 5. 选型(面试可讲)
- **为什么 MK 不用阈值比?** 阈值比是魔数、对小样本和波动敏感;MK 是非参检验,统计上有依据、抗噪。
- **为什么不用复杂时间序列(ARIMA 等)?** 数据量与场景不需要——只要"在升还是在降",MK/线性回归足矣(YAGNI)。

## 6. YAGNI
不做:季节性分解、预测/外推、变点检测。只判方向。
