# 决策日志(ADR)

> 轻量 ADR:每条记录 背景 / 决策 / 理由 / 影响。最新在前。

## ADR-007 秘密管理:本地化 + gitignore(2026-06-22)
- **状态**:Accepted
- **背景**:数据库密码、模型 API Key 多处出现。
- **决策**:`.env.local`、`.mcp.json`、`.codex/config.toml`、`private-doc/`、`001_PRD/` 全部 gitignore;`.env.example`/`.mcp-example.json` 作可提交模板;**API Key 仅存浏览器本地,永不上云、永不提交**。
- **理由**:经验库含情绪/决策/财务等私密内容;前端直连模型时 Key 在浏览器。
- **影响**:仓库零密钥;新环境需复制 example 填本地值。

## ADR-006 产品定位与路线图(2026-06-22)
- **状态**:Accepted
- **决策**:定位为 **Personal Experience Engine(个人经验管理与决策辅助引擎)**;路线图写成**能力线(提炼→关联→验证→决策→组织)×技术线**;**飞书是采集入口,不是产品**。
- **理由**:避免 PRD 发散成"AI 日记/第二大脑/飞书机器人";护城河在智能层。
- **影响**:见 `docs/version-roadmap.md`;V1+V2 为比赛核心,V5/飞书推迟。

## ADR-004 评估体系降级而非删除(2026-06-22)
- **状态**:Accepted
- **背景**:`evaluationEngine`(1714 行)+ 46 条评估用户故事,相对产品阶段过度工程化。
- **决策**:从主路径**折叠为"高级面板",保留全部代码**。

## ADR-003 经验三态 / 分类状态机拆轴(2026-06-22)
- **状态**:Accepted
- **背景**:原契约用「方向」当"能否成规则"的闸门,负向经验一律降为"待观察",丢失教训价值(与"减少重复犯错"核心矛盾)。
- **决策**:新增正交 `kind` 轴(strategy/caution/watch);迁移改用**类型 + 结构**判据;负向 + counterexample/constraint + ≥2 条件 → **避坑规则**;保留"负向伪装正向→拒绝"。
- **理由**:负向经验是最有价值的经验;`reusability` 不该兼任成熟度闸门。
- **影响**:负向沉淀为有意义的避坑规则;更新旧"非正向→watch"测试 + 新增用例。详见 `current-architecture.md` 状态机表。

## ADR-002 模型接入:前端直连 + 契约复用 + 本地兜底(2026-06-22)
- **状态**:Accepted
- **决策**:前端直连 OpenAI 兼容接口(内置 DeepSeek / 中转站 + 用户自配 Key);复用既有契约层;模型失败/网络异常**回退本地关键词引擎**(`analyzeObservationResilient`)。
- **理由**:无后端即可演示;成本可转移给用户;契约层保证模型无关且防幻觉;兜底保证任何情况下都有结构化结果、演示不崩。
- **影响**:`modelClient`/`modelConfig`/`resilientAnalysis` 三件;未配 Key 用户行为不变(走本地)。
- **权衡**:Key 在前端(已 ADR-007 约束本地化);服务端代理推迟。

## ADR-001 存储:localStorage 为真相源,云 MySQL 后置(2026-06-22)
- **状态**:Accepted
- **背景**:有一台免费 SQLPub MySQL(共享库)+ MCP 可用。
- **决策**:**5 天内 localStorage 为唯一运行时真相源**;云 MySQL 仅作开发/工具用途(`exp_` 前缀表),**不实现 app↔云同步**;同步/部署推迟。
- **理由**:契合 Local First;免后端即可演示;云库与其他项目共用,加前缀隔离。
- **影响**:`db/schema.sql` 就绪备用;未来同步靠 `exp_` 双端同构 + 幂等 upsert。
