# Experience OS 算法升级规划(本地语义引擎)

> 日期:2026-06-23 · 状态:规划(技术线,供决策/练手)
> 关联:`docs/version-roadmap.md`(V2 规律 / V4 决策)、`docs/backend-roadmap.md`(后端基建,与本文正交)、`docs/privacy-status.md`(脱敏)
> 定位:`backend-roadmap` 讲"服务端基建";本文讲 **客户端 / 离线算法**——把当前"弱关键词"的本地降级路径,升级成真算法。
> 双目标:**① 上技术强度(简历/能力);② 强化"本地优先"红线(模型不可用时也有真实智能)。** 这两件事在本项目里是同一件事。

---

## 0. 一句话动机

产品红线是"本地优先 + 模型无关":模型不可用要能离线降级。但现在的降级兜底,**全是占位级的弱关键词逻辑**。把它换成经典算法,**既上强度,又让红线真正成立**——算法在这里不是炫技,是补洞。

---

## 1. 现状:本地路径全是弱链(算法洼地)

| 位置 | 现在怎么做 | 弱在哪 |
|------|-----------|--------|
| `stores/experience.ts:inferDirection` | 几十个硬编码情绪词 | 多数真实文本→neutral(文档自承) |
| `services/aiAnalyzer.ts` | 8 条硬编码关键词规则 | 覆盖窄,无泛化 |
| `services/patternDiscovery.ts` | category/tag **精确匹配**聚类 | 同根因不同表述归不到一起 |
| `services/decisionHints.ts`(V4 召回) | 朴素分词 + 计分 | 分词烂 → 召回漂移 |

→ 这四处恰好是"无模型时"的全部智能来源。**算法升级 = 把这条弱链整体换成"分词 → 向量 → 聚类 → 召回"的本地语义引擎。**

---

## 2. 统一蓝图:本地语义引擎

```
原始观察文本
   │
   ▼  ① 中文分词(Viterbi/HMM)        ← 替换朴素分词
   ▼  ② 向量化(TF-IDF / 端上 embedding)← 新增特征层
   ├─────────────▶ ③ 离线聚类(DBSCAN/层次)  ← 替换 tag 精确匹配 → 喂 V2 规律发现
   └─────────────▶ ④ 相似召回(余弦 / ANN)   ← 替换关键词计分 → 喂 V4 决策召回
   (旁路)⑤ 进 LLM 前脱敏(Luhn+香农熵)   ← 补 privacy-status 的洞
```

**这条管线纯前端、离线可跑、数据不出设备**——与"本地优先 + 隐私"红线完全对齐。

## 4. 推荐落地顺序(按性价比)

### 阶段 A1 ⭐ 先做:中文分词(Viterbi/HMM)
**为什么先做**:解决一个**已存在的真痛**(分词烂→情绪/召回失效),且是经典 DP + 概率模型,纯 TS、离线、能讲透。一个就把"调 API 的人"洗成"会啃 NLP 的人"。

- **入门版(词典最大匹配)**:正向 / 逆向 / 双向最大匹配。需一份词典;难度低;无概率。
- **强度版(HMM + Viterbi)**:
  - 状态集 `{B, M, E, S}`(词首/词中/词尾/单字);
  - 三组概率:初始 `π`、转移 `A`(状态→状态)、发射 `B`(状态→字);
  - **Viterbi 动态规划**求最优状态序列 → 切词;
  - 概率表来源:用现成已训练表(如 jieba 数据)或在小语料上统计;**from-scratch 训练需标注语料**,演示可直接载现成表。
- **数据结构**:`dp[t][state]` + 回溯指针;时间 O(T·S²),S=4。
- **测试设计**:① 已知句子切分正确(含歧义"研究生命/研究 生命");② 未登录词不崩;③ 与朴素分词对比召回提升;④ Viterbi 回溯路径正确。
- **为什么选它**:面试问"为什么 HMM 不用规则"→"规则不处理歧义和未登录词,HMM 用概率全局最优"。

### 阶段 A2 + A3:TF-IDF + 离线聚类(DBSCAN)
**让规律发现脱离模型也能跑**,直接强化本地优先。

- **A2 TF-IDF**:分词后 → 词频 TF × 逆文档频率 IDF → 稀疏向量;相似度用余弦。
- **A3 DBSCAN**:
  - 参数 `eps`(余弦距离邻域半径)+ `minPts`(成簇最小点数);
  - 核心点 / 边界点 / **噪声点**——天然映射"复发根因(密集簇) vs 偶发(噪声)";
  - **不需预设簇数 k**(对比 K-means 的优势)。
- **备选**:层次聚类(自底向上按阈值合并,给树状图,也无需 k)。
- **测试设计**:① 同根因不同表述聚成一簇;② 干扰项被判噪声;③ 幂等(同输入同结果);④ eps 边界行为。
- **产品价值**:`patternDiscovery` 降级层从"tag 精确匹配"升级为"语义密度聚类",模型不可用时规律发现仍有效。

### 阶段 A4(可选上一档):端上 embedding 向量召回
- transformers.js 载小型多语 embedding 模型(ONNX/WASM,如 bge-small/MiniLM 量化版);
- 观察向量存 IndexedDB;决策召回用余弦 top-k;
- **卖点**:"浏览器里跑 ML、向量不出设备"——隐私 + 端上 ML 双热点。
- **代价**:模型体积(几十 MB)、首次加载、WASM 兼容——需懒加载 + 缓存。

### 阶段 A5(硬核):自实现 ANN(LSH / HNSW)
- 数据量大时余弦全量扫描慢 → 近似最近邻;
- LSH(随机超平面哈希分桶)较易;HNSW(分层可导航小世界图)较难、效果好;
- 能把 HNSW 讲清楚,就是高分项;**数据量不大时属过度优化,按需做**。

### 穿插:A6 脱敏 / A7 趋势检验(小而完整)
- **A6**:Luhn(银行卡 mod-10 校验)+ 香农熵 `H=-Σp·log₂p`(>~4 bit/char 疑似密钥)+ 邮箱/手机/身份证正则;补 `privacy-status.md` 的洞,且挂核心价值。
- **A7**:`Law.trend` 当前是阈值比 → 换 Mann-Kendall 非参趋势检验(小样本、不需正态)或时间桶线性回归。

## 6. 诚实边界(别上头)

- ❌ **别从零训模型**:环境(纯前端)不合适、性价比低。用现成概率表/小模型 + 自己写分词/聚类/召回/检索逻辑。
- ❌ **别声称"自研/研究创新"**:这是**经典算法的工程落地**,这么说够硬、经得起追问。
- ⚠️ **每个都配测试 + 一句"为什么选它"**:面试加分的是"为什么 DBSCAN 不用 K-means""为什么 HMM 不用规则",不是算法本身。
- ⚠️ **A5 是按需**:数据量小不要提前上 ANN,否则是过度工程(本项目已有 evaluationEngine 过度工程的前车之鉴)。

## 9. 通俗篇:每个算法是什么(给非技术读者 / 答辩)

整条链:`文字 →①分词→ ②变成数字 →③算相似→ ④归类/⑤召回`(旁路:⑥脱敏、⑦趋势)。
分词只是第一步(把字切成词),后面才是"让计算机会比较、会归类"。

### ① 分词 〔必需〕
把没空格的中文切成词:`周末十点健身房人少 → 周末/十点/健身房/人少`。不切,后面全失效("健身房"被切成"健/身/房"就匹配不上)。详见 `algorithm-a1-segmentation-plan.md`。

### ② TF-IDF:把词变成"带权重的数字" 〔必需〕
计算机只会比数字。TF-IDF 给每句话里的词打分:
> "周末十点**健身房**人少"——"健身房""人少"是关键词(高分),"的/了"到处都有(低分)。

把一句话变成"词的重要度数字"(向量),后面才能做数学比较。**是后面所有算法的胶水。**

### ③ 余弦相似度:算两句话有多像 〔必需〕
有了向量,算出 0~1 的相似度。0.9=很像,0.1=没关系。归类和召回都靠它。很简单。

### ④ 聚类(DBSCAN):自动把相似的归成一堆 〔必需,V2 核心〕
不用指定分几组,自己聚:
> "没开对齐会""需求没同步""没定接口" → 自动聚成 **1 条规律**;
> "今天吃了火锅"和谁都不像 → 标成**噪声**(偶发,不算规律)。

这是"规律发现"在**没有大模型时**也能跑的关键(强化"本地优先")。

### ⑤ 向量召回 / embedding:按"意思"找,不只按"字" 〔进阶,V4〕
TF-IDF 只能匹配**有相同词**的句子;embedding 按**意思**匹配:
> "没开对齐会" 和 "需求没同步"——**一个相同词都没有**,但意思接近。embedding 认得出,TF-IDF 不行。

决策召回(V4)的升级;卖点"浏览器里跑、数据不出设备"。

### ⑥ 隐私脱敏(Luhn + 香农熵):发云端前先打码 〔重要,但独立〕
不是"看懂文字",是**安全过滤**:
> - **Luhn**:校验算法,判断一串数字是不是**真银行卡号**;
> - **香农熵**:量字符串有多"乱"。API Key 很随机(熵高),正常词不会 → 揪出密钥。

补 `privacy-status.md` 那个洞(现在原文裸发云端)。

### ⑦ 趋势检验(Mann-Kendall):规律在变多还是变少 〔锦上添花〕
现在 `Law.trend` 是拍脑袋阈值比;换成统计方法,小样本也能严谨判断"这坑是不是越踩越多"。

### ⑤的提速:ANN(LSH/HNSW) 〔按需,数据多了再说〕
数据量大时召回逐条比太慢,用它加速。**现在数据少,先别做**(否则过度工程)。

### 该按什么顺序接
| 优先级 | 算法 | 作用 |
|:--:|---|---|
| 🥇 必需链 | 分词 → TF-IDF → 余弦 → **DBSCAN** | 让规律发现离线也能跑 |
| 🥈 进阶 | **embedding 召回** | 决策召回"对字"→"对意思" |
| 🥉 独立重要 | **脱敏(Luhn+熵)** | 补隐私洞,挂核心价值 |
| 可选 | 趋势检验 / ANN | 精度 / 提速,按需 |

---

> 一句话:**把那条"弱关键词"的本地降级链,换成"分词→向量→聚类→召回"的真算法——上强度、补红线、出简历金句,一次办三件事。** 先做 A1(分词),最划算。
