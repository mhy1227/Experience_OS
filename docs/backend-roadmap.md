# 后端版本规划

> 日期:2026-06-22(经 review 修订) · 状态:规划(当前处于 B0,无后端)
> 关联:产品路线图 `version-roadmap.md`;架构 `architecture/`;需求全景 `implementation-status.md`。

## 0. 现状与原则

**当前 = B0:无后端。** 纯前端 + `localStorage` + 浏览器直连模型(Key 在 `.env.local`)。demo / 单人 dogfood 够用。

原则:
- **按需引入,不按时间**:每阶段有明确触发信号;没出现该需求就不做。
- **保持薄**:后端只做前端做不了或不该做的(持 Key、批处理、向量、多人)。
- **本地优先不变**:必要前 localStorage 仍是真相源;后端是可选辅助层,不是中枢。
- **红线**:① API Key 不进浏览器;② 敏感信息进 LLM 前先脱敏(见 §1);③ 后端不擅自扩成团队/知识平台。

## 技术栈
- **Node + Hono**,部署 serverless(Vercel / CF Workers):同语言、免运维、免费档。
- 复用 `src/services/analysisContract.ts`(契约层搬后端共享)。
- DB 用 SQLPub MySQL(`db/schema.sql` 的 `exp_` 表)。Key/配置走服务端 env。

## 1. 隐私与信任(贯穿所有阶段)⭐ 新增

**问题**:产品承诺"数据只在本机"。但任何云模型(前端直连 *或* 后端代理)都会让**观察原文经过模型方/中转**。这是用云 LLM 的固有代价,必须正视。分两层应对:

### 1.1 进 LLM 前脱敏 —— 复用 `privacy-filter`
[mhy1227/privacy-filter](https://github.com/mhy1227/privacy-filter):纯 Go、毫秒级,在文本进 LLM 前剥离 **PII(邮箱/手机/身份证/银行卡含 Luhn/IP)+ 密钥(gitleaks 规则 + 上下文 + 香农熵)**,占位符不可逆。**已是 packyapi 中转的隐私合规组件**,与本项目模型通道天然契合。

集成选项(按"本地优先"优先级):
| 方案 | 在哪脱敏 | 优点 | 取舍 |
|------|----------|------|------|
| **A. 移植核心正则/熵到 TS(前端)** ⭐ | **浏览器**,发请求前 | 标识符**根本不出设备**,最强本地优先;直连/代理都受益 | 需移植 `filter/pii.go`+`secrets.go` 子集到 TS |
| B. B1 后端调用(HTTP `/redact/batch` 或 Go import) | 服务端转发模型前 | 防御纵深;复用现成服务 | 后端(Hono/Node)→ 跨语言 HTTP 调 Go 服务 |
| C. 依赖中转自带(packyapi) | 中转侧 | 零成本 | 不在自己掌控,信任前移 |

**建议**:**A 为主**(前端脱敏 = 真·本地优先),B 作后端防御纵深;不只依赖 C。

### 1.2 透明 + 分级(脱敏解决不了"语义内容仍被模型看到")
- **BYO-key 前端直连 = 隐私默认**(内容只到用户自己的模型方);
- **纯本地引擎**:完全离线、零外发(质量低,作隐私上限选项);
- **代理(B1)= 可选且透明**:明确告知"启用 = 内容会过服务器"。

> 注:privacy-filter 解决"**标识符/密钥泄露**";"模型看到经验语义"是用云 LLM 的固有取舍,靠 BYO-key + 透明 + 本地引擎分级应对,不是 filter 能消除的。

## 2. 跨切面(每阶段都适用)⭐ 新增

- **批量执行模型**:`analyze-batch` 跑几十条推理模型调用 = 数分钟,**会超 serverless 函数/CPU 上限**。→ **分块 / 流式逐条返回 / 队列**,或对大批量改用常驻运行时。**不能一个请求干完整批。**
- **测试**:后端端点要有自己的测试(沿用项目测试纪律)。
- **可观测**:日志 / 错误 / 重试 / 用量统计。
- **入口安全**:CORS、输入大小限制、入口防注入(契约层 prompt 防注入仍在)。
- **数据迁移**:DB schema 变更要有迁移(`exp_schema_meta` 已留版本位)。
- **部署/env**:Key 只在服务端 env;dev/prod 分离;本地开发故事。

---

## 3. 前后端协调机制 ⭐ 新增

后端"插进来"而非重构,靠**复用已有 `ObservationModelClient` 接缝**:
```
getActiveModelClient() → ObservationModelClient
  今天:DeepSeek 直连 client
  将来:指向 /api 的 HTTP 代理 client
analyzeObservationResilient(text,{client}) + 契约层 + store + UI  ← 全不变
```
- **后端 = 现有抽象下的一个新 client 实现**,前端编排/存储/UI 不动。
- **降级**:后端挂 / 未配 → 回退前端直连或本地引擎(沿用 resilient 模式)。后端是助攻,非单点。
- **契约共享**:前后端用同一套 `analysisContract` + `AnalysisResult`,防漂移。
- **数据归属分阶段**:B1 后端**无状态**(只处理不存),存储仍在前端;B2 才引入云存储,本地仍为主。

## 4. 阶段

### B0 · 无后端(现状)✅
前端直连 + `.env.local` + localStorage。适用 demo / 单人。**够用,不动。**

### B1 · 模型代理(薄后端)
- **触发**:Key 不能在浏览器 / 要给别人用 / 批量要快。
- **交付**:`/api/analyze`、`/api/analyze-batch`(分块或流式,见 §2);服务端持 Key、复用契约层、转发前过 privacy-filter(§1.1-B)、批处理(并行/重试/限流/缓存)。
- ⚠️ **成本/滥用**:服务端持 Key 开放他用 = **你为所有人买单 + 易被刷**。安全开放需 **BYO-key(各付各的)** 或 **B1 搭配鉴权+配额(提前引入部分 B4)**。仅"批量要快"不足以justify代理(还赔上隐私/成本),优先 BYO-key 直连。
- **体量**:S–M(端点本身)+ 配额/限流(若开放)。

### B2 · 持久化 / 云同步
- **触发**:多端 / 备份。
- **交付**:读写端点 + 增量同步。⚠️ **多端冲突/删除/合并比想象难**(接近 CRDT),按 M+ 估;先支持简单 last-write-wins + 软删除。
- 本地仍为主;DB = SQLPub `exp_` 表。

### B3 · 向量 / 语义召回
- **触发**:决策辅助要语义级(产品 V4)。
- **交付**:服务端 embeddings + 存储 + 相似检索;`exp_rules.embedding` 已预留。

### B4 · 账号 / RBAC / 多用户
- **触发**:多人 / 团队。
- **交付**:账号、鉴权、RBAC、`user_id` 隔离。
- ⚠️ **产品形态切换**,与"本地优先/不做团队协作"冲突,**做前必须重定边界**。最后、最重、最慎。

## 5. 触发条件总表(按需)

| 阶段 | 触发信号 | 不做的代价 | 体量 |
|------|----------|------------|:--:|
| B1 模型代理 | Key 不能在浏览器 / 开放他用 / 批量要快 | Key 暴露;批量慢耗钱 | S–M(+配额) |
| B2 云同步 | 多端 / 备份 | 换设备即丢 | M+ |
| B3 向量召回 | 决策辅助要语义级 | 召回停留关键词 | M |
| B4 账号/RBAC | 多人 / 团队 | 无多用户/隔离 | L |

## 6. 非目标
- 必要前不用云存储取代本地(B2 才引入,本地仍可用)。
- B1/B2 不堆账号/团队(那是 B4)。
- 不引入重框架(不迁 Nuxt、不上 Spring)——保持 Hono 薄代理。

## 7. 与产品路线图对应
| 后端阶段 | 支撑产品能力 |
|----------|--------------|
| B1 + 隐私层 | md 批量提速 + Key 安全 + 进 LLM 脱敏 |
| B3 | V4 决策辅助(语义召回) |
| B2 / B4 | 多端 / 团队(V5 生态) |

---

> 一句话:**现在不做后端;真要时起 Hono+serverless 的 `analyze-batch` 薄代理(B1),进 LLM 前用 privacy-filter 脱敏(优先移植到前端)、开放则配 BYO-key/配额、批量走分块、后端作现有 client 抽象下的可降级助攻;RBAC 最后且需重定边界。**
