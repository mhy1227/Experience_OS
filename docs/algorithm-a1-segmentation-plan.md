> 目标:用纯 TS、离线、可测的 **HMM + Viterbi** 中文分词,替换当前朴素分词,作为"本地语义引擎"的地基。

---

## 0. 目标与非目标

**做:** 纯前端、零网络的中文分词模块;两档实现(词典最大匹配 = baseline/兜底,HMM+Viterbi = 主)。
**非目标:** CRF / 神经分词 / 从零训练大模型(YAGNI,见 §9)。

**为什么是它**:当前 `decisionHints` 召回、`inferDirection` 情绪判定都依赖分词,而现状是按字符/空格的朴素切分,中文几乎失效。分词是下游所有精度(召回/聚类/情绪)的地基。

---

## 1. 模型:字位标注 HMM

把分词建模为"给每个字打标签"的序列标注问题。

- **状态集** `S = {B, M, E, S}`:词首 / 词中 / 词尾 / 单字成词。
- **观测** = 句子里的每个字符。
- **三组参数**:
  - 初始概率 `π[s]`:句首是状态 s 的概率;
  - 转移概率 `A[s_prev][s_cur]`:4×4 矩阵;合法转移有限(如 B→M、B→E、E→B、E→S、S→S、S→B…;B→S、B→B 等非法,概率置极小);
  - 发射概率 `B[s][char]`:状态 s 发出某字的概率。
- **全部用对数概率**(`log`),DP 里改乘为加,**防长句下溢**。

切词:由最优状态序列还原——遇到 `E` 或 `S` 处断词。

---

## 2. 算法:Viterbi 动态规划

```
输入:字符序列 c[0..T-1]
dp[t][s]      = 到第 t 字、状态为 s 的最大对数概率
back[t][s]    = 取得该最大值时 t-1 的状态(回溯指针)

初始(t=0):   dp[0][s] = π[s] + B[s][c0]
递推(t≥1):   dp[t][s] = max_{p} ( dp[t-1][p] + A[p][s] ) + B[s][ct]
              back[t][s] = argmax_{p}( dp[t-1][p] + A[p][s] )
终止:         best = argmax_s dp[T-1][s]
回溯:         沿 back 还原状态序列 → 按 B/M/E/S 切词
```

- **复杂度**:时间 O(T·S²)=O(16T),空间 O(T·S);S=4,极轻。
- **未登录字**:`B[s][char]` 缺失时用平滑最小概率(如 `log(1e-8)`),保证不崩、不为 -∞。

## 4. 模块设计(遵守 services 分层)

`src/services/segmentation.ts`(纯函数,无 DOM,可测):

```ts
// 仅签名示意,非实现
export interface SegProbs { pi: ...; trans: ...; emit: ... }      // 对数概率
export function segmentByHMM(text: string, probs: SegProbs): string[]
export function segmentByDict(text: string, dict: Set<string>): string[] // 兜底/baseline
export function segment(text: string): string[]                    // 入口:HMM 优先,异常回退 dict/按字
```

- 概率表作为静态 JSON 资源**懒加载**(不进主 bundle 关键路径)。
- 入口 `segment` 内置降级:HMM 概率表加载失败 → 词典最大匹配 → 再不行按字符切。**与项目"降级链"哲学一致。**

## 5. 预处理 / 边界

- **英文/数字**:连续 ASCII 字母数字整体成 token(不拆),直接旁路 HMM。
- **标点/空格**:作分隔符,不进词。
- **空串 / 纯标点**:返回 `[]`,不崩。
- **混合文本**("开 PR review 会"):中文段走 HMM,英文段整体保留。
- **歧义经典例**:"研究生命的起源"(研究/生命 vs 研究生/命)、"结婚的和尚未结婚的"——用于测试。

## 10. 面试可讲点(选型理由)

- **为什么不用纯规则/词典?** 规则不解决歧义与未登录词;词典最大匹配是贪心、非全局最优。
- **为什么 HMM+Viterbi?** 用转移+发射概率做**全局最优**序列标注,DP 保证最优解,复杂度 O(16T) 极低,纯前端可跑。
- **为什么不上 CRF/神经?** 数据量与场景不需要;过度工程(项目已有 evaluationEngine 的前车之鉴)。
- **下溢怎么处理?** 全程对数概率,乘改加。

## 附:分词是啥(大白话)

**分词 = 把一句连续的中文,切成一个个有意义的"词"。**

英文有空格天然分好(`I go to gym`);中文挤一起,计算机看到的只是一串字符,不知道哪到哪是一个词:

```
周末十点健身房人少   →   周末 / 十点 / 健身房 / 人少
```

**为什么重要**:召回、情绪、聚类全是**按词**工作的。不分词、按字瞎切,"健身房"会被切成"健/身/房",匹配全失效。

**难点(为什么算个真算法)**:同一句能有多种切法,得判断哪种最可能对:

```
研究生命的起源
  研究 / 生命 / 的 / 起源   ✅
  研究生 / 命 / 的 / 起源   ❌
```

HMM+Viterbi 就是"用概率挑出最可能那种切法"的经典方法。
