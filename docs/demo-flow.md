# Experience OS 演示流程指南

> 用途：现场演示操作手册
> 日期：2026-06-30

---

## 一、演示前准备（5 分钟）

### 1.1 环境检查

| 检查项 | 操作 | 标准 |
|--------|------|------|
| 网络 | 打开浏览器访问网络 | 正常 |
| AI 模型 | 检查 API Key 是否配置 | 可用 |
| 浏览器缓存 | 清空 localStorage | 干净状态 |
| 本地服务 | 启动 `npm run dev:h5` | 端口 5173 |

### 1.2 数据准备（两种方式）

#### 方式一：批量导入演示数据（推荐）

1. 打开页面，点击「数据管理」
2. 从 `docs/demo-data.md` 复制数据
3. 粘贴到批量导入框，点击导入
4. 导入完成后，点击「扫描我的 90 天」生成规律

#### 方式二：现场实时输入（更有说服力，但风险高）

> 注意：现场输入依赖 AI 模型响应，建议作为"加分项"而非核心流程

1. 清空所有数据，保持干净状态
2. 演示过程中逐句输入观察

## 二、演示流程（核心闭环，4-5 分钟）

### 阶段 1：AI 提炼（1.5 分钟）

**目标**：展示"一句话输入 → 结构化规则"的核心能力

**操作步骤**：

1. **清空数据**（可选，展示从零开始）
   > "我们从干净状态开始，让大家看到整个流程。"

2. **输入第一条观察**
   ```
   下午3点写方案效率最高
   ```
   > "用户只需说一句口语化的话，不需要任何结构化模板。"

3. **点击「生成规则」**
   > "系统会自动调用 AI 进行分析。"

4. **展示 AI 产出**
   - 类别：工作
   - 方向：策略(strategy)
   - 标题：高效写作时段策略
   - 结论：下午3点左右是方案写作的高效时段
   - 推荐行动：将重要写作任务安排在下午3点
   > "AI 自动完成分类、提炼结论、给出推荐行动。用户完全不需要手动整理。"

5. **再输入一条负向观察**（可选）
   ```
   上午10点开会容易走神
   ```
   > "同样的流程，负向观察会被归类为'避坑规则'。"

---

### 阶段 2：规律发现（1.5 分钟）

**目标**：展示"跨记录聚类 → 规律库"的能力

**操作步骤**：

1. **提前准备好多条观察**（或现场快速输入）
   ```
   下午3点写方案效率最高
   上午10点开会容易走神
   晚上加班写代码容易出错
   ```

2. **点击「扫描我的 90 天」**
   > "系统会自动扫描所有记录，发现相似观察并聚合成规律。"

3. **展示规律库结果**
   - 规律："工作效率与时段强相关"
   - 复发次数：3次
   - 趋势：稳定
   - 置信度：中
   > "多条表面不同但同根因的观察，系统会自动聚合成一条规律。这条规律告诉我们，工作效率和时段有很强的相关性。"

4. **展示规律生命周期**（可选）
   - 点击「标记已复盘」
   > "规律有生命周期，可以标记'已复盘'或'已解决'。"

---

### 阶段 3：决策召回（1 分钟）

**目标**：展示"主动召回 + 可信度建议"的能力

**操作步骤**：

1. **输入新的观察**
   ```
   明天要写方案
   ```

2. **展示召回结果**
   > "系统主动召回了之前记录的'下午3点写方案效率最高'，并给出建议。"

3. **展示可信度信号**（如果有）
   - 可信度：已验证
   - 建议：安排在下午3点
   > "基于历史验证记录，系统给出了可信度标注和建议。"

---

### 阶段 4：总结与收尾（30 秒）

**操作步骤**：

1. **回顾核心闭环**
   > "回顾一下整个流程：一句话观察 → AI 自动提炼 → 规律发现 → 决策时主动召回。"

2. **强调价值**
   > "核心价值是：让每一次经历，都变成下一次决策的资产。"

3. **开放 Q&A**
   > "以上就是 Experience OS 的核心能力，欢迎提问。"

## 三、现场插入数据的策略

### 3.1 核心原则

**不建议现场全部实时输入**，原因：
- AI 模型响应不稳定（网络延迟、API 限流）
- 现场压力大，容易出错
- 演示时间有限（4-5 分钟）

**推荐策略：混合模式**

| 数据类型 | 处理方式 | 原因 |
|----------|---------|------|
| 种子数据（5-8 条） | 提前批量导入 | 保证规律发现有足够样本 |
| 演示关键步骤 | 现场实时输入 | 展示真实操作流程 |
| 决策召回触发 | 现场输入 | 展示主动召回能力 |

### 3.2 演示数据准备清单

提前准备好以下数据（从 `docs/demo-data.md` 中筛选）：

**工作效率主线（3 条，用于规律发现）**：
- 下午3点写方案效率最高
- 上午10点开会容易走神
- 晚上加班写代码容易出错

**其他类别（各 1-2 条，展示分类能力）**：
- 周末上午十点去健身房，人最少
- 工作日晚上八点去超市，结账几乎不用排队
- 睡前一小时不看手机，入睡明显更快

### 3.3 现场输入话术

**输入第一条时：**
> "我们从一句简单的观察开始。"

**输入后续时：**
> "再记一条..."

**输入触发召回时：**
> "现在假设我面临一个决策..."

## 四、常见问题与应对

### Q：AI 模型响应慢怎么办？

**应对**：
1. 提前准备好备用截图
2. 如果等待超过 10 秒，切换到备用演示（录屏视频）
3. 同时解释："这是网络延迟，正常使用时响应很快"

### Q：规律发现没有产出预期结果怎么办？

**应对**：
1. 提前验证演示数据，确保能产出规律
2. 如果现场失败，展示提前准备好的截图
3. 解释："这是因为样本量不够，实际使用中积累一定数据后效果会更好"

### Q：决策召回没有命中怎么办？

**应对**：
1. 提前验证触发词和召回数据的匹配度
2. 如果现场失败，手动展示规则库中的相关规则
3. 解释："召回基于语义匹配，我们正在优化匹配算法"

### Q：数据安全问题？

**应对**：
1. 展示「隐私脱敏」功能（如果已实现）
2. 说明数据存储在浏览器本地（localStorage）
3. 说明 API Key 不上传服务器

## 五、演示兜底方案

### 5.1 录屏视频（首选）

- 提前录制 3 分钟完整演示视频
- 包含所有核心场景
- 如果现场翻车，播放视频

### 5.2 备用截图

- 每个场景的关键界面截图
- 按顺序编号
- 如果网络中断，展示截图讲解

### 5.3 离线降级演示（可选）

- 断开网络连接
- 展示本地关键词分析引擎的工作效果
- 说明："即使模型不可用，系统仍能通过本地算法正常工作"

## 六、时间控制建议

| 环节 | 时长 | 控制技巧 |
|------|------|---------|
| 讲解 PPT | 3 分钟 | 掐时间演练，避免超时 |
| Demo 阶段 1（提炼） | 1.5 分钟 | 提前准备好数据，现场只输入一条 |
| Demo 阶段 2（规律） | 1.5 分钟 | 提前导入数据，现场只做扫描操作 |
| Demo 阶段 3（召回） | 1 分钟 | 快速输入触发词 |
| 总结收尾 | 0.5 分钟 | 简洁有力 |
| Q&A | 2 分钟 | 控制回答时间 |

## 八、备注

- 演示时保持语速适中，给评委理解时间
