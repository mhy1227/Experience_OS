# Experience OS 产品介绍

> 撰写日期：2026-06-29
> 用途：项目介绍、反问应对、内部宣讲材料

## 一、项目起源：为什么需要 Experience OS

### 1.1 一个真实的困惑

你有没有过这样的经历：

- **记了等于没记**：在备忘录里写了一句"11点去买面包最划算"，三个月后完全想不起来
- **试了等于白试**：上次踩坑的教训，下一次换了个场景又踩一遍
- **知道但提取不出来**：隐约感觉有些规律，但说不清楚，也没法复用

这不是记忆力的问题，而是**工具的失配**。

我们现有的工具要么太轻（记完就忘），要么太重（知识库建了三年没填几页）。Experience OS 想填补这个空白。

### 1.2 核心问题：经验为什么留不下来？

| 问题 | 原因 | 现有工具的回应 |
|------|------|----------------|
| 记了忘 | 没有和下一次决策挂钩 | 笔记：无 |
| 难复用 | 缺乏结构化提炼 | 知识库：需要人工维护 |
| 区分不了偶然和规律 | 没有时间维度和样本积累 | 无 |

Experience OS 的切入点是：**不让你做额外的工作，让 AI 自动完成从"观察到经验"的转化**。

## 二、这不是什么：为什么不是知识库和笔记系统

### 2.1 笔记系统的局限

**笔记的本质是存储，不是处理。**

- 写完一条笔记，它就躺在那里
- 你需要主动去翻，主动去想这条和哪条相关
- 没有提示，没有关联，没有生命周期

> 笔记是仓库，但仓库不会自动整理。

### 2.2 知识库的局限

**知识库的前提是你有时间、有意愿去系统化整理。**

- 需要你主动建立分类体系
- 需要定期维护和更新
- 门槛太高，大部分人坚持不下来

> 知识库是图书馆，但大多数人不是馆长。

### 2.3 Experience OS 的定位

```
笔记（太轻）-------- Experience OS -------- 知识库（太重）
    |                      |                      |
  随便记                   AI帮你整理             主动维护
  记完就忘                 下次自动召回            需要坚持
```

**Experience OS = 笔记的零门槛 + 知识库的智能化**

它不需要你建分类、不需要你维护、不需要你定期打开。它只需要你**偶尔记录一句话**，其余的由系统完成：

- 自动提炼成规则
- 自动关联相似记录
- 自动发现趋势和规律
- 在需要时主动提示你

## 三、面对反问的回应

### Q1：这个和普通的笔记有什么区别？

**反问**：我用一个备忘录 App 不是一样吗？

**回应**：
备忘录记完就忘，下次想不起来。Experience OS 会主动帮你提炼和关联：
- 你记一句"工作日早上地铁挤"，系统提炼出"早8点前到公司可避开早高峰"
- 你下次规划出行时，系统主动提醒你

本质上，备忘录是**静态存储**，Experience OS 是**动态处理**。

---

### Q2：这个和 Notion/飞书知识库有什么区别？

**反问**：Notion 也能建笔记库，为什么还要这个？

**回应**：
Notion 需要你主动维护——建页面、贴标签、定期整理。大部分人的 Notion 最后变成了"收藏夹里的僵尸页面"。

Experience OS 让你**零维护**：
- 不需要建页面
- 不需要贴标签
- 不需要定期整理
- 系统自动提炼、自动关联、自动提醒

---

### Q3：这不就是 AI 总结吗？大模型不是也能做？

**反问**：ChatGPT 也能总结我的文字啊？

**回应**：
大模型的总结是**一次性的**，不会沉淀。

Experience OS 会**持续追踪**：
- 你记了5次"周末健身房人少"
- 系统发现这是**规律**而不是偶然
- 后续会自动提升置信度，形成稳定的"周末低峰训练"规则
- 下次你计划周末健身，系统主动提醒你

这不是一次性的总结，而是**经验资产的积累**。

---

### Q4：我的经验真的值得被"系统化"吗？

**反问**：我又不是什么大人物，积累经验有什么用？

**回应**：
经验的价值不在于"大"，而在于**重复**。

你以为只有"商业决策"才需要经验，其实：
- "哪家外卖周六晚高峰送得最快"是经验
- "什么天气适合晾被子"是经验
- "和某个供应商沟通的最佳时机"是经验

这些小事积累起来，每年能帮你省下不少"试错成本"。

---

### Q5：隐私安全怎么保证？

**反问**：我把生活细节都记在这里，数据安全吗？

**回应**：
Experience OS 设计了三层隐私保护：

1. **本地优先**：数据默认存在浏览器 localStorage，不上传到任何服务器
2. **敏感信息过滤**：系统自动识别并过滤敏感信息（账号密码、银行卡号等）
3. **可选后端**：如果需要多端同步，数据经用户自己的后端代理，系统不接触明文数据

---

### Q6：这个和"复盘"有什么区别？

**反问**：敏捷开发里不是有复盘(Sprint Retrospective)吗？

**回应**：
复盘是**事件驱动**的——需要你先经历一个完整的项目或阶段。

Experience OS 是**观察驱动**的：
- 不需要等一个项目结束
- 不需要开复盘会议
- 只需要**随时记录一句话**
- 系统帮你积累、提炼、关联

复盘是定期大盘点，Experience OS 是持续的小记账。

---

### Q7：如果 AI 提炼错了怎么办？

**反问**：AI 的分析万一不准确呢？

**回应**：
系统有三层保障：

1. **可纠正**：每条提炼结果你都可以修正
2. **可验证**：通过"复测"机制，让时间验证规律的真伪
3. **可追溯**：所有变更都有历史记录

AI 是助手，不是权威。你始终是经验的主人。

## 四、技术实现要点

### 4.1 核心流程

```
用户输入（一句话观察）
    ↓
AI 结构化分析（提取主题、时间、地点、对象、结论）
    ↓
安全契约校验（防注入、防幻觉、三态判定）
    ↓
分类入库（规则库 / 待观察 / 反例池）
    ↓
规律发现（跨记录聚类，发现趋势）
    ↓
决策召回（下次遇到类似场景，自动提示）
```

### 4.2 关键设计原则

| 原则 | 说明 |
|------|------|
| **模型优先，本地兜底** | 优先调用 AI 模型，异常时降级到本地关键词引擎 |
| **契约层校验** | 所有 AI 输出必须经过安全校验，不能直接写入 |
| **三态判定** | 每条记录判定为：正向规则 / 反例 / 待观察 |
| **隐私优先** | 敏感信息自动过滤，数据本地存储 |
| **可干预** | 用户可以修正任何 AI 提炼结果 |

## 五、产品边界

### 5.1 Experience OS 能做什么

- 接收一句话观察，自动提炼结构化规则
- 跨记录发现相似经验和趋势规律
- 在合适的时机主动召回相关经验
- 支持复测机制，逐步提升规则置信度
- 支持批量导入历史记录
- 支持数据导出（Markdown / JSON / CSV）

### 5.2 Experience OS 不能做什么

- 不能替代专业的知识管理系统
- 不能保证每条提炼都100%准确（需要用户参与校验）
- 不能自动处理复杂的多步骤工作流
- 不能提供实时数据（如天气预报、交通状态）

## 七、结语

Experience OS 想解决的是一个很小但很痛的问题：**我们的经验为什么总是留不下来？**

不是因为我们懒得记，而是因为现有的工具都在要求我们做更多——建分类、写总结、定期整理。

Experience OS 的目标是：**让你用最少的动作，获得最大的收益**。

只需要随手记一句话，剩下的交给系统。
